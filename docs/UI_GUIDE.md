# Witness Me UI Guide

Witness Me는 한글, 영어 등 사용자가 기억해야 하거나 알고 싶은 표현을 검색하고, 뜻과 해석을 확인한 뒤, 기억할 포인트를 저장해 알림 퀴즈로 다시 복습하는 언어 복습 앱이다.

핵심 경험은 `찾는 순간 이해하고, 잊기 전에 다시 묻는다`이다. 사용자는 긴 학습 세션을 시작하지 않아도 된다. 궁금한 문장, 문장 속 숙어와 단어, 단어/숙어를 바로 검색하고, 앱은 복습할 만한 포인트를 짧고 명확하게 정리해 준다.

## Product Goal

- 한국어, 영어 등 다국어 문장과 표현을 자연스럽게 검색하게 한다.
- 검색한 문장, 단어, 숙어의 뜻과 문맥상 해석을 빠르게 이해하게 한다.
- 사용자가 기억해야 할 포인트를 낮은 입력 부담으로 저장하게 한다.
- 저장한 포인트를 지정된 시간에 알림과 퀴즈로 다시 꺼내 복습하게 한다.
- 복습 결과를 바탕으로 저장소에서 표현의 상태와 기억 안정도를 확인하게 한다.

## Core Flow

핵심 플로우는 다음 순서를 유지한다.

1. 검색
2. 뜻/해석 확인
3. 기억 포인트 저장
4. 알림 설정
5. 퀴즈 복습

### 1. 검색

사용자는 홈에서 기억하고 싶은 표현을 바로 입력한다.

- 검색 대상은 문장, 문장 속 숙어/단어, 단어, 숙어를 모두 포함한다.
- 입력 예시는 다국어 가능성을 보여줘야 한다.
- 사용자가 언어를 먼저 고르게 하지 않는다.
- 검색창은 홈의 주인공이어야 하며, 주변 기능은 최소화한다.
- 빈 상태에서는 기능 설명보다 `무엇을 검색하면 되는지`를 알려준다.

좋은 검색 힌트:

- `예: I’m down for it / 손이 많이 가다 / take off`
- `문장, 단어, 숙어를 어떤 언어로든 검색해 보세요`
- `궁금한 표현을 붙여넣으면 뜻과 기억 포인트를 정리해요`

### 2. 뜻/해석 확인

검색 결과는 빠른 이해를 우선한다.

- 결과 첫 화면에는 가장 가능성 높은 뜻과 문맥상 해석을 먼저 보여준다.
- 긴 설명보다 `뜻`, `문맥`, `예문`, `주의할 점`을 짧게 나눈다.
- 문장 검색 결과는 전체 해석과 핵심 표현을 함께 보여준다.
- 단어/숙어 검색 결과는 대표 뜻과 실제 사용 뉘앙스를 구분한다.
- 불확실하거나 여러 의미가 가능한 경우에는 `문맥에 따라 달라질 수 있어요`처럼 솔직하게 안내한다.

### 3. 기억 포인트 저장

저장은 학습 기록 작성이 아니라 복습 카드 만들기여야 한다.

- 사용자는 검색 결과에서 기억할 포인트를 선택하거나 바로 저장할 수 있어야 한다.
- 기본 저장 포인트는 앱이 제안하고, 사용자는 필요한 경우만 수정한다.
- 한 카드에는 하나의 기억 포인트만 담는다.
- 저장 버튼은 결과 화면의 주요 액션으로 배치한다.
- 저장 완료 후에는 다음 행동을 두 가지 이하로 제안한다.

저장 완료 후 좋은 액션:

- `알림 설정하기`
- `저장소에서 보기`
- `계속 검색하기`

### 4. 알림 설정

알림은 사용자가 복습할 수 있는 실제 시간에 맞춘다.

- 알림 메뉴에서는 시간과 퀴즈 개수를 한 세트로 관리한다.
- `아침`, `점심`, `퇴근길` 같은 고정 루틴 라벨을 기본 강요하지 않는다.
- 사용자가 직접 시간을 정하고, 필요하면 여러 알림을 추가한다.
- 알림 추가 버튼은 폼을 여는 역할만 한다.
- 제출 버튼을 눌렀을 때만 새 알림을 만든다.
- 각 알림은 켜짐/꺼짐, 시간, 문제 수, 예상 소요 시간, 삭제 액션을 명확히 보여준다.

기본 복습량 옵션:

- `짧게` 5문제 · 약 3분
- `가볍게` 10문제 · 약 5분
- `집중` 20문제 · 약 10분

### 5. 퀴즈 복습

퀴즈는 짧고 예측 가능해야 한다.

- 알림을 열면 바로 오늘의 퀴즈를 시작할 수 있어야 한다.
- 한 화면에는 하나의 질문만 보여준다.
- 문제는 저장한 기억 포인트를 중심으로 낸다.
- 정답 피드백은 즉시 제공한다.
- 틀린 경우에는 비난하지 않고 다시 기억할 힌트를 준다.
- 결과 화면은 맞힌 개수, 다시 볼 표현, 다음 복습 시간을 간단히 보여준다.

## Navigation

하단 메뉴는 네 가지로 정리한다.

- 홈
- 복습
- 알림
- 저장소

### 홈

검색과 저장 진입의 시작점이다.

- 중앙 검색 입력을 가장 강하게 보여준다.
- 최근 검색이나 추천 표현은 보조 정보로만 사용한다.
- 복습 통계, 알림 설정, 저장소 목록을 홈에 과하게 노출하지 않는다.
- 직접 추가가 필요하다면 홈의 보조 액션으로 제공하되 별도 탭으로 만들지 않는다.

### 복습

지금 풀 수 있는 퀴즈와 최근 복습 결과를 보여준다.

- 주요 액션은 `퀴즈 시작`이다.
- 오늘 예정된 복습이 있으면 문제 수와 예상 시간을 보여준다.
- 퀴즈가 없으면 저장소에서 복습할 표현을 고를 수 있게 안내한다.
- 성취 표현은 짧게 유지하고, 과한 게임 UI로 만들지 않는다.

### 알림

복습 시간과 문제 수를 관리한다.

- 알림 추가, 편집, 켜기/끄기, 삭제를 이 화면에 모은다.
- 시간과 문제 수가 분리되어 보이지 않게 한다.
- 스위치 상태는 텍스트와 시각 상태가 함께 전달되어야 한다.
- 알림 권한이 꺼져 있으면 이유와 해결 액션을 명확히 안내한다.

### 저장소

검색 후 저장한 기억 포인트의 목록이다.

- 저장된 표현, 짧은 뜻, 마지막 복습 상태를 보여준다.
- 문장, 단어, 숙어 타입은 필요할 때만 보조 태그로 보여준다.
- 사용자는 저장소에서 항목을 다시 열어 뜻, 예문, 퀴즈 기록을 확인할 수 있다.
- 오래 안 본 항목과 자주 틀리는 항목은 부드럽게 다시 복습으로 연결한다.

## Apple HIG UX Principles

### Clarity

모든 화면은 사용자가 지금 할 일을 즉시 이해할 수 있어야 한다.

- 화면당 주요 액션은 하나로 유지한다.
- 버튼명은 결과를 설명해야 한다. 예: `저장하기`, `퀴즈 시작`, `알림 추가`
- 추상적인 제품 용어보다 사용자의 언어를 쓴다.
- 문장, 단어, 숙어의 차이는 필요할 때만 설명한다.

### Immediacy

검색과 피드백은 즉각적으로 느껴져야 한다.

- 검색 입력 후 로딩 상태를 명확히 보여준다.
- 결과가 나오면 뜻/해석이 화면 상단에 바로 보여야 한다.
- 저장 성공, 알림 생성, 정답 여부는 지연 없이 피드백한다.
- 사용자가 다음 단계로 이동하기 전에 불필요한 확인 단계를 늘리지 않는다.

### Low Input Burden

사용자에게 학습 카드를 작성하라고 요구하지 않는다.

- 언어 선택, 타입 선택, 긴 설명 입력은 기본 요구사항에서 제외한다.
- 앱이 기억 포인트를 제안하고 사용자는 확인하거나 짧게 수정한다.
- 알림 설정은 시간과 복습량만으로 끝나야 한다.
- 기본값은 안전하게 제안하되 사용자가 쉽게 바꿀 수 있어야 한다.

### Accessibility

접근성은 시각 보조 기능이 아니라 기본 사용성의 일부다.

- 현재 탭은 `aria-current="page"`로 표시한다.
- 검색 입력, 퀴즈 선택지, 알림 스위치에는 명확한 접근성 이름을 제공한다.
- 색상만으로 정답/오답이나 켜짐/꺼짐 상태를 구분하지 않는다.
- 텍스트 대비는 라이트/다크 모드 모두에서 충분해야 한다.
- Dynamic Type 또는 브라우저 글자 크기 확대 시 카드 내용이 잘리지 않아야 한다.
- 스크린 리더 사용자가 검색 결과의 뜻, 예문, 저장 액션 순서를 자연스럽게 따라갈 수 있어야 한다.

### Touch Targets

모바일에서 한 손으로 안정적으로 조작할 수 있어야 한다.

- 주요 터치 대상은 최소 44px에 가깝게 유지한다.
- 하단 탭, 퀴즈 선택지, 저장 버튼, 알림 스위치는 충분한 간격을 둔다.
- 삭제처럼 되돌리기 어려운 액션은 다른 주요 액션과 붙여 두지 않는다.
- safe area를 고려해 하단 메뉴와 주요 버튼이 기기 가장자리에 너무 붙지 않게 한다.

### Restrained Motion

모션은 이해를 돕는 용도로만 사용한다.

- 화면 전환은 방향성과 맥락을 알려주는 정도로 짧게 사용한다.
- 정답/오답 피드백은 과한 흔들림이나 번쩍임을 피한다.
- 배경 장식 모션은 최소화하고 텍스트 가독성을 해치지 않는다.
- `prefers-reduced-motion` 사용자는 장식 모션과 불필요한 전환 효과를 끈다.

## Visual Direction

디자인 방향은 `Apple HIG inspired language utility`이다.

- 밝고 차분한 그룹 배경과 흰색 카드 소재를 기본으로 한다.
- 주요 액션은 Apple 시스템 블루 계열을 사용한다.
- 성공, 오류, 경고 색상은 의미 기반으로 제한해 사용한다.
- 텍스트와 여백을 통해 신뢰감을 만들고, 과한 장식으로 학습 앱처럼 무겁게 만들지 않는다.
- 글래스 효과는 보조적으로만 사용하고 대비를 해치지 않는다.
- 다크 모드는 시스템 설정에 자연스럽게 반응한다.

## Typography

- 기본 폰트는 Apple 시스템 폰트 스택을 사용한다.
- 제목은 짧고 기능적인 문장으로 쓴다.
- 검색 결과의 표현 원문은 가장 눈에 띄게 보여준다.
- 뜻과 해석은 본문 크기에서도 읽기 쉽게 충분한 줄 간격을 둔다.
- 영어, 한국어, 기타 언어가 섞여도 줄바꿈이 어색하지 않게 카드 폭과 행간을 잡는다.
- 버튼과 탭은 한눈에 이해되는 한국어 메뉴명을 우선한다.

## Content Guidelines

문구는 다국어 표현 검색이 가능하다는 느낌을 주되, 과장하지 않는다.

### Voice

- 친절하고 간결하게 쓴다.
- 사용자를 평가하지 않는다.
- `학습 관리`보다 `다시 보기`, `기억하기`, `퀴즈로 확인하기`처럼 일상적인 표현을 쓴다.
- AI를 전면에 내세우기보다 사용자의 이해와 복습을 돕는 도구로 설명한다.

### Good Copy

- `문장, 단어, 숙어를 어떤 언어로든 검색해 보세요`
- `뜻과 문맥을 확인하고 기억할 포인트만 저장해요`
- `이 표현을 퀴즈로 다시 물어볼게요`
- `알림 시간마다 저장한 표현을 짧게 복습해요`
- `틀린 표현은 다음 복습에 한 번 더 보여드릴게요`
- `예: I’m down for it / 손이 많이 가다 / take off`

### Avoid

- `AI 기반 개인화 지식 관리 시스템`
- `Expression Intelligence Dashboard`
- `학습 데이터를 기반으로 최적화된 리마인드 경험을 제공합니다`
- `출근길 / 점심 / 퇴근길` 같은 고정 루틴 강요
- `현재 선택: 08:10 · 10분 · 5문제`처럼 사용자가 결정하기 전의 상태를 확정처럼 보이는 문구
- 사용자를 탓하는 오답 문구. 예: `틀렸습니다`, `아직 외우지 못했어요`

## Screen Guidelines

### Home

홈 화면 구성:

- 검색 입력
- 검색 버튼 또는 제출 액션
- 다국어 예시 힌트
- 최근 저장 또는 최근 검색으로 돌아가는 작은 보조 액션

홈 화면에 두지 말아야 할 것:

- 긴 앱 소개 문구
- 복습 통계 전체
- 알림 설정 폼
- 저장소 전체 목록
- 여러 개의 경쟁하는 CTA

### Search Result

검색 결과 구성:

- 원문 표현
- 뜻 또는 전체 해석
- 문맥상 뉘앙스
- 예문 또는 사용 상황
- 기억 포인트 제안
- `저장하기` 주요 액션

결과 화면은 사용자가 `이걸 저장할지` 판단할 수 있을 만큼만 보여준다. 깊은 설명은 접을 수 있는 보조 영역으로 둔다.

### Save Point

기억 포인트 저장 구성:

- 저장할 표현
- 기억 포인트 한 줄
- 퀴즈에서 물어볼 방식 미리보기
- 저장 버튼

사용자가 직접 수정할 수 있는 영역은 짧게 유지한다. 긴 메모 입력을 기본으로 열어두지 않는다.

### Review

복습 화면 구성:

## Interaction Rules

- 하단 탭을 누르면 홈, 복습, 알림, 저장소 패널로 이동한다.
- 홈 검색 제출 후 검색 결과 화면으로 이동한다.
- 검색 결과에서 `저장하기`를 누르면 기억 포인트가 저장된다.
- 저장 완료 후 사용자는 알림 설정, 저장소 보기, 계속 검색 중 하나로 이동할 수 있다.
- `알림 추가`는 추가 폼을 여는 역할만 한다.
- 알림 폼에서 제출 버튼을 눌렀을 때만 알림이 생성된다.
- 알림 스위치는 실제 체크박스 또는 동등한 접근성 상태를 가져야 한다.
- 퀴즈 선택지를 누르면 정답 여부와 짧은 설명이 즉시 표시된다.
- 퀴즈 결과에서 다시 볼 표현은 저장소 상태에 반영된다.

## Responsive Behavior

- 콘텐츠 영역은 화면 폭에 맞춰 중앙 정렬한다.
- 홈 검색창은 너무 넓게 늘어나지 않도록 최대 폭을 제한한다.
- 좁은 화면에서는 검색 입력과 버튼이 자연스럽게 쌓이거나 줄어든다.
- 하단 탭은 safe area를 고려한다.
- 퀴즈 선택지는 엄지로 누르기 쉬운 간격을 유지한다.
- 알림 카드는 세로로 쌓여도 시간, 문제 수, 스위치가 잘리지 않아야 한다.
- 긴 문장과 다국어 텍스트는 카드 밖으로 넘치지 않고 줄바꿈되어야 한다.

## Do / Don't

Do:

- 홈은 다국어 표현 검색에 집중한다.
- 뜻/해석을 먼저 보여주고, 상세 설명은 뒤로 둔다.
- 기억 포인트는 앱이 제안해 입력 부담을 낮춘다.
- 알림은 시간과 문제 수를 한 세트로 보여준다.
- 퀴즈는 짧고 즉각적인 피드백을 준다.
- 메뉴명은 홈, 복습, 알림, 저장소로 유지한다.

Don't:

- 홈에 모든 기능을 한꺼번에 노출하지 않는다.
- 사용자가 언어, 타입, 설명을 모두 직접 입력하게 만들지 않는다.
- 저장 카드를 긴 노트처럼 만들지 않는다.
- `알림 추가` 클릭만으로 알림을 만들지 않는다.
- 오답 피드백을 평가나 실패처럼 느끼게 만들지 않는다.
- 과한 모션, 네온 컬러, 게임 UI로 복습 경험을 무겁게 만들지 않는다.
